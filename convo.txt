🖥️ What “Main Display” Actually Means in Windows

When you connect two or more displays (e.g., a laptop screen + external monitor), Windows treats them as separate screens in a single workspace.

You can move windows between them, extend the desktop, or duplicate one onto another.

Among those, Windows always designates one of them as the “Main Display.”

⚙️ The Main Display Controls 4 Key Things
Feature	What It Means
🧭 Taskbar and Start Menu	The main display is where your taskbar, Start button, and system tray appear by default.
🪟 New Windows / Apps	Most applications open on the main display first (unless Windows remembers a previous position).
💻 Desktop Icons	All your desktop icons appear on the main display by default.
🔆 Primary Focus	Windows treats it as your primary workspace, and most system notifications appear there.
📡 Does “Main Display” Affect Projection or Screen Sharing?

Not directly — but it influences what is being shared depending on the mode you choose:

1. Duplicate Mode

Both screens show the same thing (main display’s content).

Here, it doesn’t matter much which one is “main,” because both mirror each other.

2. Extend Mode

The screens become one big workspace (e.g., Screen 1 + Screen 2).

If you project or share your screen (e.g., in Zoom or SpaceDesk),
Windows or your app might default to sharing the “Main Display” — unless you manually pick the other one.

So yes, if you set Display 2 as Main Display, then:

The taskbar moves there.

Most apps open there.

And screen-sharing software (including SpaceDesk, Zoom, OBS, etc.) will often treat it as the one to project by default.

🧭 Quick Analogy

Think of your main display as your “control center.”
It’s where your desktop lives — and where your computer “thinks” your primary work happens.
The other monitors just extend or mirror it.

Since your app is for song presentation, here’s what you’d typically do:

Run the app on your laptop (main screen).

const { BrowserWindow, screen } = require('electron');

const displays = screen.getAllDisplays();
const externalDisplay = displays.find(d => d.id !== screen.getPrimaryDisplay().id);

if (externalDisplay) {
  const projectorWindow = new BrowserWindow({
    x: externalDisplay.bounds.x,
    y: externalDisplay.bounds.y,
    fullscreen: true,
    frame: false,
    backgroundColor: '#000000',

}


This opens your presentation view (lyrics, etc.) on the second display, while your main control interface stays on your laptop.
